# Semestr 5

## Technologie Backendowe

Przedmiot koncentruje się na tworzeniu warstwy serwerowej aplikacji internetowych.

Główne zagadnienia:

- JavaScript po stronie serwera,
- Node.js i npm,
- asynchroniczność,
- protokół HTTP/HTTPS,
- obsługa formularzy, sesji i middleware,
- dokumentowe bazy danych,
- operacje CRUD, indeksowanie i replikacja,
- projektowanie i implementacja REST API,
- bezpieczeństwo i testowanie backendu.

Po ukończeniu przedmiotu student powinien potrafić zaprojektować i stworzyć backend aplikacji internetowej oraz połączyć go z bazą danych i warstwą frontendową.

---

## Technologie Frontendowe

Przedmiot dotyczy tworzenia nowoczesnych interfejsów użytkownika w aplikacjach internetowych.

Główne zagadnienia:

- HTML i CSS,
- preprocesory CSS,
- responsywność i dostępność,
- architektura SPA,
- React i JSX,
- zarządzanie stanem aplikacji,
- formularze i walidacja,
- routing,
- integracja z backendem,
- testowanie i optymalizacja aplikacji.

Po ukończeniu przedmiotu student powinien potrafić zaprojektować i zaimplementować interaktywną aplikację frontendową oraz zintegrować ją z usługami backendowymi.

---

# Semestr 6

## Technologie DevOps

Przedmiot obejmuje przygotowanie aplikacji do uruchamiania, wdrażania i utrzymania w środowisku produkcyjnym.

Główne zagadnienia:

- wirtualizacja i konteneryzacja,
- Docker i Dockerfile,
- obrazy, kontenery, wolumeny i sieci,
- Docker Compose,
- debugowanie i monitorowanie kontenerów,
- Continuous Integration i Continuous Delivery,
- Jenkins,
- automatyzacja wdrożeń za pomocą Ansible,
- wdrażanie aplikacji w chmurze,
- monitoring aplikacji produkcyjnej.

Po ukończeniu przedmiotu student powinien potrafić skonteneryzować aplikację, zautomatyzować proces jej budowania i wdrażania oraz uruchomić ją w środowisku chmurowym.

---

# Semestr 7

## Trendy w rozwoju API

Przedmiot koncentruje się na aktualnych kierunkach projektowania, implementacji i zarządzania interfejsami API.

Główne zagadnienia:

- rola API w nowoczesnych aplikacjach internetowych,
- przegląd stylów i technologii API,
- REST, SOAP, GraphQL i gRPC,
- dobre praktyki projektowania REST API,
- porównanie różnych architektur API,
- dokumentacja API,
- OpenAPI, Swagger i Postman,
- TypeScript,
- bezpieczeństwo, skalowalność i wydajność API.

Po ukończeniu przedmiotu student powinien potrafić dobrać odpowiedni styl API do potrzeb projektu, zaprojektować nowoczesny interfejs API oraz przygotować jego dokumentację.

---

# Podsumowanie

Specjalizacja rozwija kompetencje w pełnym cyklu tworzenia aplikacji internetowych:

1. projektowanie interfejsu użytkownika,
2. tworzenie warstwy backendowej,
3. integracja z bazą danych,
4. komunikacja przez REST API,
5. testowanie aplikacji,
6. konteneryzacja,
7. automatyzacja wdrożeń,
8. utrzymanie aplikacji w środowisku produkcyjnym.
